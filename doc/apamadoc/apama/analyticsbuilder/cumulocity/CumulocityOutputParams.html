apama.analyticsbuilder.cumulocity
Event CumulocityOutputParams

Parameters for declaring Cumulocity events produced by an output block.

The combination of event type, field values, and device identifies the output stream.

This is typically used as following to declare the output stream and then save the output handler created: 
        

        CumulocityOutputHandler outputHandler;
        

        

        action $validate() returns Promise {
        

        	CumulocityOutputParams params := CumulocityOutputParams.forSyncEventType($parameters.deviceId, self, Event.getName(), fields);
        

        	return params.declare(saveHandler);
        

        }
        

        action saveHandler(CumulocityOutputHandler handler) {
        

        	self.outputHandler := handler;
        

        }
The declare action performs an inventory lookup for the provided device and then declares the output stream being produced using the BlockBase.consumesInput action.

declare

            apama.analyticsbuilder.Promise declare(action<apama.analyticsbuilder.cumulocity.CumulocityOutputHandler> handlerCreated)
Declare the output stream being produced by the block.

This is an asynchronous operation that performs an inventory lookup for the target device. It will call the handlerCreated action after the lookup is done.

It will call BlockBase.producesOutput() after the lookup to declare that the block is an output block.

This must be called during the $validate action, not afterward. Either the promise returned by this action or another promised chained with it must be returned from the $validate action.
Parameters:
handlerCreated - The Callback action to save the output handler created.

Create a new CumulocityOutputParams for asynchronous output events.

Asynchronous output events are events that do not have source timestamps and can only be consumed by another model in a time-asynchronous manner when they are received back from Cumulocity IoT. Examples of asynchronous output events are Operation and ManagedObject.

forSyncEventType

            apama.analyticsbuilder.cumulocity.CumulocityOutputParams static forSyncEventType(any deviceId, any block, string eventTypeName, dictionary<string, any> fields)
Create a new CumulocityOutputParams for synchronous output events.

Synchronous output events are events that have source timestamps and can be consumed by another model in a time-synchronous manner and can be processed by the model with any other data from the same timestamp. Examples of synchronous output events are Alarm, Event, and Measurement.

Synchronous output events are routed internally for consumption by any consumer model before being sent to Cumulocity IoT. These output events might be received back at some point in the future. In such cases, the value will already have been processed and so must be ignored. This is achieved by adding a tag to the output event when it is sent to Cumulocity IoT. Input blocks should check for the presence of such a tag and ignore an input event if the tag is present. Output Cumulocity events are tagged automatically if CumulocityOutputHandler is used for sending output events.
Parameters:
deviceId - The target device identifier or the Trigger Device.
block - The block object.
eventTypeName - The fully qualified name of the Cumulocity event that block produces. The events sent by the block should be of this type only.
fields - The field values that the block sends.
Returns:
A new CumulocityOutputParams.
