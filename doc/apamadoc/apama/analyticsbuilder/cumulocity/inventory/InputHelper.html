apama.analyticsbuilder.cumulocity.inventory
Event InputHelper

Deprecated:
This helper event has been deprecated, use the CumulocityInputParams and CumulocityInputHandler events instead.
Helper methods for input blocks.

Helpers for input blocks to process InventoryLookupResults, calling listensForInput and recording the c8y_inputs on the model scope parameters. This is typically used as so: 

InputHelper ihelper := InputHelper.forBlock(self, $modelScopeParameters);
ihelper.setInput($parameters.deviceId, Measurement.getName(), {"type":$parameters.measurementType, "fragment":fragment, "series":series});


return InventoryLookup.lookup($parameters.deviceId).andThen(ihelper.declareInput);
(with other callbacks before ihelper.declareInput if needed). The block should have a throwNoDevices action which throws a suitable exception for no devices found (without this, a less useful exception will be reported).
See Also:
CumulocityInputParams - To declare an input block.
CumulocityInputHandler - To schedule an input event for processing.

Promise callback for declaring input.

This will call BlockBase.listensForInput and set the c8y_inputs in modelScopeParameters. If the block has isGroup or devices members, they are set from the InventoryLookupResult.
Parameters:
a_lookupResult - The promised result, which should be an InventoryLookupResult.
