apama.analyticsbuilder.cumulocity
Event CumulocityInputParams

Parameters for declaring Cumulocity events consumed by an input block.

The combination of event type, field values, and device identifies the input stream.

This is typically used as following to declare the input stream and save the input handler created: 
        

        CumulocityInputHandler inputHandler;
        

        

        action $validate() returns Promise {
        

        	CumulocityInputParams params := CumulocityInputParams.create($parameters.deviceId, self, Event.getName()).withFields({"type":$parameters.type});
        

        	return c8yInputParams.declare(inputHandlerCreated);
        

        }
        

        action saveHandler(CumulocityInputHandler handler) {
        

        	self.inputHandler := handler;
        

        }
The declare action performs an inventory lookup for the provided device and then declares the input stream being consumed using the BlockBase.consumesInput action.

declare

            apama.analyticsbuilder.Promise declare(action<apama.analyticsbuilder.cumulocity.CumulocityInputHandler> handlerCreated)
Declare the input stream being consumed by the block.

This is an asynchronous operation that performs an inventory lookup for the source device or device group. It will call the handlerCreated action after the lookup is done.

It will call BlockBase.consumesInput() after the lookup to declare that the block is an input block.

This must be called during the $validate action, not afterward. Either the promise returned by this action or another promised chained with it must be returned from the $validate action.
Parameters:
handlerCreated - The Callback action to save the input handler created.
